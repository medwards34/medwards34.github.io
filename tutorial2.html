Add files to a Github repository

Githubs main purpose is using repositories to store, edit, and change programs. Since editing is one of the features Github allows, you will often upload files to existing projects. Below is a guide that will demonstrate how to add local files onto an existing repository.

    Upload files

    Choose files

    Documents

    Files description

    Picking Branches & committing




    Main page

  1. Navigate over to the main page of the repository you want files to be uploaded from. The example below will be uploading files to the "How-To-Add-Files-Tutorial" repository.

    2. Upload files

  2. Click the “Add file” drop down box and choose “Upload files”.

    3. Choose files

  3. You can either drag and drop your files into the box or click to choose the files you want to upload. In this example the files are being chosen by the “choose your files” link.

    4. Documents

  4. Find the file or files you want to upload and add it to the repository.

    5. Files description

  5. Once you’ve chosen the files, you will see the files name uploaded below the drag and drop box.

  6. Below the uploaded file you will be prompted to add a description of the file you just uploaded.

    7. Picking branches & committing

  7. Before committing the change, you can either add this file to the main branch of your repository or you can create a new branch and start a pull request to your repository. This example will commit to the main branch of the repository.

  8. Click the “Commit Changes” button and the file will be committed to your repository.
